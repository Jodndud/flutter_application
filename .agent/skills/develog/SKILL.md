# 목적

어제의 커밋을 기능 단위로 분석해 읽기 좋은 작업 로그로 재구성한다.

# 절차

1. `git log --since="yesterday 00:00" --until="today 00:00" --oneline` 실행
2. 각 커밋의 diff를 확인 (`git show <hash> --stat`)
3. 관련 커밋끼리 기능 단위로 묶어서 정리

# 출력 규칙

- 커밋 메시지를 그대로 나열하지 말 것
- 무엇을 만들었고 왜 그렇게 했는지 서술형으로 작성

# 출력 형식

## {날짜} 작업 로그
### 만든 것
{기능/작업 단위 서술}
### 기술적으로 주목할 점
{설계 결정, 트레이드오프 등}
### 열린 PR
{PR 목록 또는 "없음"}
